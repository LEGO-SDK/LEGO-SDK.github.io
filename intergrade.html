集成

SDK 可以拆分成两个独立的部分，JavascriptBridge 负责构建交互通道，Modules 则是官方提供的 API。
你可以独立集成 JavascriptBridge 而无需集成 Modules，接下来，你可以自行编写 API。
你也可以只集成 Modules 中的某一个 API 而无需担心过多的 API 影响应用的稳定性。

iOS

安装 CocoaPods

SDK 使用 CocoaPods 进行依赖管理，集成 SDK 前，你需要安装 CocoaPods。
具体的使用说明请参照 https://guides.cocoapods.org/using/getting-started.html。 
$ sudo gem install cocoapods

全部集成

这种方式会将 JavascriptBridge 以及所有API集成到目标应用

1. 添加依赖配置到 Podfile

pod 'LEGO-SDK'

2. 更新依赖代码

$ pod update

单独集成

这种方式会将 JavascriptBridge 以及选定的 API 集成到目标应用

1. 选择需要集成的模块

JavascriptBridge
Native
UI
WebView

1. 添加以下配置到 Podfile

pod 'LEGO-SDK/Core'

2. 更新依赖代码

pod update

Android

Gradle

LEGO SDK 的 Gradle 仓库位于 JitPack，因此你需要在顶部目录 build.gradle 文件中添加仓库。 
allprojects {
    repositories {
        jcenter()
        maven { url "https://jitpack.io" }
    }
}

单独集成

这种方式将使用 Gralde 集成 JavascriptBridge 以及选定的 API 集成到目标应用

1. 选择需要集成的模块

JavascriptBridge
Native
UI
WebView

1. 添加以下配置到 build.gradle

compile 'com.github.LEGO-SDK.LEGO-SDK-Android:core:0.3.0'

2. Sync

如果使用 Android Studio，IDE 会提示你同步依赖。
